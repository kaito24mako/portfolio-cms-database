# Sequelize Model Options (SQLite)

When defining columns in your Sequelize models, you have access to many powerful options beyond just the data type. These options control constraints, defaults, relationships, and more.

## Importing Required Modules

To define models, you need to import DataTypes and your database connection:

```javascript
const { DataTypes } = require("sequelize");
const sequelize = require("../util/db");
```

---

## Default Values

Default values are automatically assigned to a column when no value is provided during record creation.

```javascript
const User = sequelize.define("User", {
	isActive: {
		type: DataTypes.BOOLEAN,
		defaultValue: true, // New users are active by default
	},
	role: {
		type: DataTypes.STRING,
		defaultValue: "user", // Default role is 'user'
	},
	createdAt: {
		type: DataTypes.DATE,
		defaultValue: DataTypes.NOW, // Automatically set to current timestamp
	},
});
```

**Special Default Values**:

- **`DataTypes.NOW`**: Automatically sets the current date/time when a record is created
- **Static values**: Any fixed value (strings, numbers, booleans)
- **Functions**: Can use Sequelize functions for dynamic defaults

---

## Null Constraints

The `allowNull` option controls whether a column can contain `NULL` values.

```javascript
const Product = sequelize.define("Product", {
	name: {
		type: DataTypes.STRING,
		allowNull: false, // Required field - cannot be NULL
	},
	description: {
		type: DataTypes.TEXT,
		allowNull: true, // Optional field - can be NULL (this is the default)
	},
});
```

**Key Points**:

- **`allowNull: false`**: Adds `NOT NULL` constraint - database will throw an error if you try to insert NULL
- **`allowNull: true`**: Column can be NULL (this is the default behavior)
- **Default**: If not specified, `allowNull` defaults to `true`

**Example Use Case**:

```javascript
// Email is required, phone is optional
const Contact = sequelize.define("Contact", {
	email: {
		type: DataTypes.STRING,
		allowNull: false, // Must provide an email
	},
	phone: {
		type: DataTypes.STRING,
		allowNull: true, // Phone is optional
	},
});
```

## Unique Constraints

The `unique` option ensures that no two records can have the same value in that column.

### Simple Unique Constraint

```javascript
const User = sequelize.define("User", {
	username: {
		type: DataTypes.STRING,
		unique: true, // No two users can have the same username
	},
	email: {
		type: DataTypes.STRING,
		unique: true, // No two users can have the same email
	},
});
```

### Composite Unique Constraints

Multiple columns can form a composite unique key by using the same string identifier:

```javascript
const Enrollment = sequelize.define("Enrollment", {
	studentId: {
		type: DataTypes.INTEGER,
		unique: "student_course", // Part of composite key
	},
	courseId: {
		type: DataTypes.INTEGER,
		unique: "student_course", // Part of composite key
	},
});
// A student can only enroll in each course once
// But the same studentId or courseId can appear in different combinations
```

**Use Cases**:

- **Simple unique**: Usernames, email addresses, product SKUs
- **Composite unique**: Prevent duplicate combinations (student+course, user+product rating)

## Primary Keys

Primary keys uniquely identify each record in a table.

```javascript
const Product = sequelize.define("Product", {
	sku: {
		type: DataTypes.STRING,
		primaryKey: true, // Use SKU as primary key instead of default 'id'
	},
	name: DataTypes.STRING,
	price: DataTypes.DECIMAL(10, 2),
});
```

**Note**: By default, Sequelize automatically creates an `id` column as the primary key. You only need to specify `primaryKey: true` when using a custom primary key.

**Example with UUID Primary Key**:

```javascript
const User = sequelize.define("User", {
	id: {
		type: DataTypes.UUID,
		defaultValue: DataTypes.UUIDV4,
		primaryKey: true, // UUID as primary key
	},
	username: DataTypes.STRING,
});
```

---

## Auto Increment

Auto-incrementing columns automatically generate sequential numbers for new records.

```javascript
const Order = sequelize.define("Order", {
	orderNumber: {
		type: DataTypes.INTEGER,
		autoIncrement: true,
		primaryKey: true,
	},
	customerName: DataTypes.STRING,
	total: DataTypes.DECIMAL(10, 2),
});
// Orders will be numbered: 1, 2, 3, 4, 5...
```

**Requirements**:

- Must be used with `INTEGER` or `BIGINT` types
- Often combined with `primaryKey: true`
- Values are automatically generated by the database

## Custom Column Names

The `field` option maps your model attribute name to a different database column name.

```javascript
const User = sequelize.define("User", {
	firstName: {
		type: DataTypes.STRING,
		field: "first_name", // Database column is 'first_name'
	},
	lastName: {
		type: DataTypes.STRING,
		field: "last_name", // Database column is 'last_name'
	},
});

// In your code: user.firstName
// In database: first_name column
```

**Benefits**:

- Use camelCase in JavaScript while maintaining snake_case in database
- Work with legacy databases that have different naming conventions
- Keep code clean and idiomatic for each context

## Foreign Keys

Foreign keys establish relationships between tables by referencing the primary key of another table.

```javascript
const Order = sequelize.define("Order", {
	userId: {
		type: DataTypes.INTEGER,
		references: {
			model: "Users", // Name of the target table
			key: "id", // Column in the target table
		},
	},
	productId: {
		type: DataTypes.INTEGER,
		references: {
			model: "Products",
			key: "id",
		},
	},
	quantity: DataTypes.INTEGER,
});
```

**Foreign Key Options**:

- **`model`**: The name of the referenced model/table (can be string or model object)
- **`key`**: The column name in the referenced table (usually "id")

**Example with Model Object**:

```javascript
const Category = sequelize.define("Category", {
	name: DataTypes.STRING,
});

const Product = sequelize.define("Product", {
	categoryId: {
		type: DataTypes.INTEGER,
		references: {
			model: Category, // Reference the model directly
			key: "id",
		},
	},
	name: DataTypes.STRING,
});
```

## Complete Example: Combining Multiple Options

Here's a real-world example that combines multiple column options:

```javascript
const BlogPost = sequelize.define("BlogPost", {
	id: {
		type: DataTypes.UUID,
		defaultValue: DataTypes.UUIDV4,
		primaryKey: true,
	},
	title: {
		type: DataTypes.STRING,
		allowNull: false,
		unique: true, // No duplicate titles
	},
	slug: {
		type: DataTypes.STRING,
		allowNull: false,
		unique: true,
		field: "post_slug", // Database column name
	},
	content: {
		type: DataTypes.TEXT,
		allowNull: false,
	},
	status: {
		type: DataTypes.ENUM,
		values: ["draft", "published", "archived"],
		defaultValue: "draft",
	},
	authorId: {
		type: DataTypes.INTEGER,
		allowNull: false,
		references: {
			model: "Users",
			key: "id",
		},
	},
	viewCount: {
		type: DataTypes.INTEGER,
		defaultValue: 0,
	},
	publishedAt: {
		type: DataTypes.DATE,
		allowNull: true, // Only set when published
	},
});
```

## Summary: Model Options Quick Reference

### `defaultValue`

**Purpose:** Set default value for new records  
_Example:_ `defaultValue: true`

### `allowNull`

**Purpose:** Control NULL/NOT NULL constraint  
_Example:_ `allowNull: false`

### `unique`

**Purpose:** Ensure uniqueness (single or composite)  
_Example:_ `unique: true` or `unique: "composite_key"`

### `primaryKey`

**Purpose:** Designate as primary key  
_Example:_ `primaryKey: true`

### `autoIncrement`

**Purpose:** Auto-generate sequential numbers  
_Example:_ `autoIncrement: true`

### `field`

**Purpose:** Map to different database column name  
_Example:_ `field: "first_name"`

### `references`

**Purpose:** Create foreign key relationship  
_Example:_ `references: { model: "Users", key: "id" }`
